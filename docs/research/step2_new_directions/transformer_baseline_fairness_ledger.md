# Transformer Baseline Fairness Ledger

Date frozen: `2026-05-07`.

This ledger freezes validation-only baselines and ablations for the Step 2 replay-capped advantage-memory transformer. The adjacent JSON file is the machine-readable source of truth generated by `benchmarks/step2_transformer_baseline_fairness_ledger.py`.

The primary candidate remains `advantage_post_ffn_memory` with primary metrics `final_window_nll` and `eval_nll` at horizons `3000`, `5000`, and `10000`. Perplexity, placement, fast-only metrics, gate diagnostics, and compute are secondary. No entry below may be used to retune candidate flags after validation.

## Lockbox Rule

Run validation first. For single-config entries, lockbox uses the same frozen config. For validation-grid baselines, validation selects exactly one config using the predeclared selection rule; only that selected config is run on lockbox. The lockbox wrapper command must include `--validation-decision-summary` pointing to a validation summary whose `clears_confirmatory_bar` is exactly `true`; otherwise it refuses before split materialization. Direct-runner lockbox commands in the JSON are plan-only entries, not permission to bypass the wrapper. Do not inspect lockbox to choose widths, learning rates, placement, gate caps, replay settings, or reporting metrics.

## Required For Paperworthiness

| Entry | Family | Status | Why it is required |
|---|---|---|---|
| `optimizer_matched_ffn_sgd` | baseline | runnable now | Required primary comparator. Without it there is no paired claim against the tuned FFN transformer. |
| `parameter_matched_ffn_h96` | baseline | runnable now | Required to rule out the 2080 extra prototype value parameters as the cause. |
| `wider_ffn_h128` | baseline | runnable now | Required because the observed memory margins are small and could be a width effect. |
| `state_matched_ffn_control` | baseline | hook needed | Required if the paper claims an algorithmic memory effect rather than a state or update-budget effect. |
| `wall_clock_matched_ffn` | baseline | hook needed | Required because the replay path is materially slower than the FFN path. |
| `no_replay_current_gate` | mechanism_ablation | runnable now | Required to attribute the result to delayed validation-like utility rather than any advantage gate. |
| `no_cap_replay_gate` | mechanism_ablation | runnable now | Required to show the positive result is not hiding an uncapped memory failure mode. |
| `fixed_gate_cap_only` | mechanism_ablation | runnable now | Required to isolate learned gate adaptation from a static residual ensemble. |
| `placement_pre_ffn_kv` | mechanism_ablation | from primary results | Required as a secondary check. It must not replace the frozen post-FFN primary method after validation or lockbox outcomes are known. |
| `fast_only_memory_training_attribution` | attribution | from primary results | Required to state whether the mechanism is inference-time memory or training-time auxiliary regularization. |
| `fast_loss_only_training_ablation` | attribution | runnable now | Required for training-attribution claims; secondary for the primary performance claim. |
