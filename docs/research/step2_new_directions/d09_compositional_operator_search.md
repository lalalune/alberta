# D09 Compositional Operator Search

## Purpose

D07 left one hard Step 2 blocker: `synthetic_compositional`. The best arccosine/NNGP KRLS run reached about 0.3725 final-window MSE at budget 256, while the fair MLP baseline was about 0.2758. Residual MLP+kernel hybrids did not fix this.

D09 attacks that blocker directly with fixed nonlinear operators plus an online linear readout. This is not an MLP router, not an MLP residual, and not a trained trunk. Each candidate initializes a nonlinear observable map at birth and updates only the readout on every time step with normalized LMS:

`w_h <- w_h + eta * error_h * phi(x) / (1 + ||phi(x)||^2)`.

The mathematically relevant analogy is an online extreme learning machine / random-feature operator: a fixed basis approximates a nonlinear function class, and the temporal learner handles continual readout adaptation.

## What Actually Worked

The one-layer random tanh operator worked. This is not surprising after seeing the target class: the stream is generated by tanh compositions, and a broad fixed tanh dictionary gives the linear readout enough nonlinear coordinates to track the current context.

The simpler one-layer map beat the deeper random tanh map. The deeper random operator was almost tied with MLP but did not robustly beat it. In this online setting, extra random composition increases feature variance and does not help the readout adapt faster.

Step size matters, but not delicately. Width 256 with eta `0.4` and eta `0.6` both beat MLP on all three seeds. Eta `0.2` was too slow and landed at `0.276191`, essentially the MLP boundary.

Width is not the critical ingredient once enough random tanh features exist. Width 128, 256, and 512 all beat MLP. Width 512 was best, but width 256 is the cleaner canonical bank because it keeps readout dimension to 257 while still winning `3/0/0`.

Fourier, Chebyshev tensor, and Volterra features did not solve this blocker. They are plausible for frequency/polynomial regimes, but they are the wrong inductive bias for a random tanh-composition oracle.

## Compute

The winning random-tanh operators are cheaper than a trained MLP in update complexity because only the output readout is learned. For width `m` and `H` heads, each step costs one fixed projection `O(m * d)`, one tanh, and a readout update `O(m * H)`.

Measured runtime in the focused 3-seed run:

- `tanh_lms_w256_eta0p4_s1`: about `0.645 s` mean runtime per seed.
- `tanh_lms_w512_eta0p4_s1`: about `0.768 s`.
- `mlp_h128`: about `0.844 s`.
- `mlp_h64`: about `1.225 s`.

These timings include JAX compilation effects, so they should not be treated as exact microbenchmarks. The direction is still useful: random-tanh readout learning is not buying accuracy by spending substantially more computation than the fair MLP.

## Spillover Generalization

After beating MLP on `synthetic_compositional`, the winner family was tested on `controlled_frequency` and `digits_label_drift`.

### Controlled Frequency

Best MLP:

- `mlp_h64_64`: final-window MSE `0.156857`.

Random-tanh operators:

- `tanh_lms_w256_eta0p4_s1`: final-window MSE `0.563880`.
- `tanh_lms_w512_eta0p4_s1`: final-window MSE `0.564236`.

Best-candidate versus best-MLP diff: `-0.399718 +/- 0.029240`, wins/losses/ties `0/3/0`.

Conclusion: random tanh features are not the frequency solution. This agrees with D07: raw degree-3 polynomial structure is the right tool for the controlled frequency task because the target is exactly cubic in `sin(theta)`.

### Digits Label Drift

Best MLP:

- final-window MSE `0.038250` from `mlp_h64`.
- best held-out test accuracy `0.9041` from `mlp_h128`.

Random-tanh operators:

- `tanh_lms_w512_eta0p4_s1`: final-window MSE `0.046290`, held-out test accuracy `0.8776`.
- `tanh_lms_w256_eta0p4_s1`: final-window MSE `0.047786`, held-out test accuracy `0.8627`.

Best-candidate versus best-MLP:

- final-window MSE diff `-0.008153 +/- 0.001182`, wins/losses/ties `0/3/0`.
- test accuracy diff `-0.029066 +/- 0.004460`, wins/losses/ties `0/3/0`.

Conclusion: the random-tanh operator is not a stateful external classifier solution by itself. It should be a compositional bank inside a larger resource-managed learner, not the full Step 2 learner.

## Assessment

D09 resolves the narrow compositional blocker: a standalone fixed random-tanh operator with normalized LMS readout beats the best fair MLP on `synthetic_compositional` under paired 3-seed testing.

D09 does not resolve Step 2 alone. Its failure on controlled frequency and label-drift digits is useful because it sharpens the canonical design: Step 2 needs a multi-bank learner, not a single operator. The random-tanh bank should be retained for tanh-compositional targets; the raw polynomial/RKHS banks from D07 should cover polynomial/frequency and stateful retention regimes.

The canonical candidate from this file is:

`tanh_lms_w256_eta0p4_s1`

The higher-scoring but larger candidate is:

`tanh_lms_w512_eta0p4_s1`

For the final Step 2 learner, the right move is to add this random-tanh bank to the D07 multi-bank/resource-manager path and let the resource manager allocate update budget across polynomial, algebraic/retention RKHS, and random-tanh operator banks. This is the first clean non-router evidence that the compositional MLP advantage can be beaten by a cheaper fixed operator.
